# Brief And Structure

Use this reference when the user has not provided enough presentation detail or when you need a fast slide outline.

## Minimum Brief

Ask only for the missing high-impact details:

1. What product, project, or topic is this deck about?
2. Who is the audience?
3. What tone should it have?

Helpful but optional:

4. Preferred brand color or reference image
5. Slide count
6. Existing content, screenshots, or notes

If the brief is still thin after that, proceed with sensible assumptions and state them.

## Default Deck Shapes

### Internal status deck

Use for internal syncs, management updates, or project reviews.

1. Cover
   Project name, presenter, date, agenda
2. Core topic A
   Demand, problem, signal, or current status
3. Core topic B
   Solution, version plan, roadmap, or decision framing
4. Next step
   Action plan, owners, decisions needed

### Product or pitch deck

Use for investors, customers, launches, or product storytelling.

1. Cover
   Product name and one positioning sentence
2. Problem
   Three-column pain points or market gaps
3. Solution
   Core product idea, product pillars, or feature cards
4. Roadmap
   Timeline, milestones, or staged rollout
5. Core belief
   Differentiator, traction signal, or key metrics
6. Closing
   Summary and CTA

## Content Expansion Rules

When the user gives too little content, strengthen the deck instead of leaving generic placeholders.

- Problem slide:
  Add industry context, user quotes, quantified pain, or workflow friction.
- Solution slide:
  Add technical explanation, product mechanics, or competitive framing.
- Roadmap slide:
  Add versions, milestones, dates, and dependencies.
- Next-step or closing slide:
  Distill concrete asks, decisions, owners, or CTA language.

## Tone Mapping

- Formal business:
  Tight headlines, restrained copy, clear decision framing.
- Internal relaxed:
  More direct, conversational labels, lighter copy density.
- Data-driven:
  Metrics, charts, bars, and count-up treatments should carry more weight than narrative slogans.

## Structure Heuristics

- Keep each slide centered on one message.
- Prefer `3` columns, `2`-panel splits, or a single hero composition.
- If slide count is not specified, default to `4-6`.
- If the audience is senior, bias toward decisions, trade-offs, and next actions.
- If the audience is external, bias toward narrative clarity and proof.
